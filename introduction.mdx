<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-3xl font-bold text-gray-900 dark:text-white mb-4">
    Quick start
  </h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
    Get up and running with Dagster in minutes
  </p>

  <Steps>
    <Step title="Install Dagster">
      Install Dagster using pip or uv:

      ```bash
      pip install dagster dagster-webserver
      ```

      Or with uv:

      ```bash
      uv add dagster dagster-webserver
      ```
    </Step>

    <Step title="Define your first asset">
      Create a Python file with a simple data asset:

      ```python my_data_pipeline.py
      import dagster as dg
      import pandas as pd

      @dg.asset
      def my_dataset() -> pd.DataFrame:
          return pd.DataFrame({
              "id": [1, 2, 3],
              "name": ["Alice", "Bob", "Charlie"]
          })
      ```

      Assets are the core building blocks of Dagster. Each asset represents a data object that you want to build and maintain.
    </Step>

    <Step title="Create a Definitions object">
      Define your code location by wrapping assets in a `Definitions` object:

      ```python my_data_pipeline.py
      import dagster as dg
      import pandas as pd

      @dg.asset
      def my_dataset() -> pd.DataFrame:
          return pd.DataFrame({
              "id": [1, 2, 3],
              "name": ["Alice", "Bob", "Charlie"]
          })

      defs = dg.Definitions(
          assets=[my_dataset]
      )
      ```
    </Step>

    <Step title="Launch the Dagster UI">
      Start the Dagster web server to visualize and execute your pipeline:

      ```bash
      dagster dev -f my_data_pipeline.py
      ```

      Navigate to `http://localhost:3000` to see your asset graph and materialize your assets.

      <Note>
        The Dagster UI provides a visual interface for exploring your data lineage, monitoring runs, and debugging issues.
      </Note>
    </Step>
  </Steps>
</div>
